Le projet

Lors de mon stage au sein de l’association ICATY, j’ai participé au développement d’une application mobile et web complète dans un contexte réel de situation client/développeur. Ma mission principale était de concevoir l’application « Un jour internet », de la conception jusqu’au déploiement. J’ai travaillé sur le back-end avec la création d’une API et la gestion de la base de données, ainsi que sur le front-end mobile et le back-office web destiné aux administrateurs. Cette application a pour objectif de sensibiliser les personnes âgées aux usages du numérique et de renforcer leur sécurité en ligne. Elle propose un mot par jour, accompagné d’une explication simple et accessible.

Architecture de l’application

L’application « Un jour internet » est structurée en plusieurs parties afin d’assurer une expérience fluide, sécurisée et évolutive.

Front-end mobile ( pour les utilisateurs )

Destiné aux utilisateurs, il permet de consulter le mot du jour, d’accéder aux anciens mots, de choisir l’heure des notifications et de contacter ICATY. Flutter

Back-office web ( pour les administrateurs )

Interface réservée aux administrateurs authentifiés, permettant la gestion des mots, des comptes administrateurs ainsi que l’accès aux statistiques de l’application. Html / CSS / JavaScript

Back-end & base de données

Le back-end gère la logique métier, le traitement des requêtes et la communication avec la base de données, qui centralise les mots, les utilisateurs et les paramètres de l’application. Python Flask SQLite

Fonctionnalités principales

  • Réception d’une notification quotidienne à heure définie
  • Accès au mot du jour et à ses explications
  • Personnalisation de l’heure des notifications
  • Posibilité de revoir les mots des jours précedent
  • Application gratuite financée par la publicité
  • Gestion complète des contenus par les administrateurs
  • Statistiques anonymisées d’utilisation accesible seulement pour les administrateurs

Conception & architecture technique

Conception de la base de données

La base de données a été conçue en utilisant la méthode Merise afin de structurer efficacement les données et garantir leur cohérence. J’ai utilisé l’outil Looping pour réaliser les modèles conceptuels et logiques (MCD / MLD / MPD), ce qui m’a permis de formaliser l’architecture de la base avant son implémentation.

Modèle Conceptuel de Données (MCD) réalisé avec Looping

Portabilité du projet

J’ai également utilisé Docker afin de faciliter le déploiement et garantir la portabilité de l’application sur différents environnements. Cette approche m’a permis d’assurer une configuration stable et reproductible du projet, indépendamment de la machine utilisée.
